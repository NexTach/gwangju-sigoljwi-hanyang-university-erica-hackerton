# Road DNA 센서 보정 가이드

## 목적

MVP 임계치는 접근성 표준이나 장애물 분류 모델이 아니다. 동일한 기기를 실제
휠체어·유모차 프레임에 고정하고 반복 주행해, 평지 진동과 이동 충격 후보를
구분할 수 있는 탐색값을 찾는 절차다.

## 기본 조건

- 휴대폰을 프레임에 단단히 고정하고 화면 방향과 장착 위치를 기록한다.
- 휠체어·유모차·보행 데이터는 각각 별도 세션으로 수집한다.
- GPS 정확도 25m 이하, 속도 0.25m/s 이상에서만 후보를 수용한다.
- 원본 센서 스트림은 서버로 보내지 않는다. 2초 창에서 계산한 평균, 표준편차,
  최대 피크, 피크 수, RMS와 후보만 전송한다.
- `LOW < MEDIUM < HIGH < DROP` 순서를 항상 유지한다.

## 현장 절차

1. 디버그 빌드에서 홈의 `센서 보정` 화면을 연다.
2. 평탄한 구간을 같은 속도로 최소 10회 왕복하고 최대 피크와 RMS 범위를 적는다.
3. 작은 이음매, 거친 노면, 실제 턱을 각 10회 이상 통과한다.
4. 기기를 손으로 한 번 치거나 떨어뜨리는 단발 패턴도 별도 기록한다.
5. 평지 상위 95%보다 높은 지점을 `LOW`, 반복적으로 구분되는 경계를
   `MEDIUM/HIGH`, 단발 극단치를 `DROP` 후보로 둔다.
6. 값을 저장하고 눈가림 재주행으로 오탐·미탐을 확인한다.
7. 기기 모델·장착 위치·타이어·노면·속도·임계치·결과를 함께 기록한다.

기본 탐색값은 다음과 같으며 확정값이 아니다.

| 항목                   |        값 |
| ---------------------- | --------: |
| Low impact peak        |  1.8 m/s² |
| Medium impact peak     |  3.5 m/s² |
| High impact peak       |  5.5 m/s² |
| Possible drop peak     | 22.0 m/s² |
| Repeated vibration RMS | 1.15 m/s² |

## 판정 원칙

- 한 번의 좌표를 턱·포트홀로 확정하지 않는다.
- 큰 단발 피크는 `HELD_DROP_PATTERN`으로 보류한다.
- GPS 정확도가 낮으면 `HELD_LOW_GPS_ACCURACY`, 정지 상태면
  `REJECTED_STATIONARY`로 점수에서 제외한다.
- 여러 익명 기여자의 반복성과 최신성을 신뢰도에 반영한다.
- 임계치를 바꾼 모든 시연은 앱 버전과 함께 기록해 비교 가능하게 한다.
