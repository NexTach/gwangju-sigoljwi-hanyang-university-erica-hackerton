# 테스트 전략

기본 테스트 스위트는 핵심 로직에 대한 빠르고 결정적인 단위 테스트만 실행한다.
테스트 이름과 구조는 `Describe → Context → It` 순서를 사용한다.

## 실행

```bash
pnpm test

(cd apps/mobile && flutter test)
(cd packages/road_dna_design && flutter test)
```

정적 검증까지 포함하려면 웹·API는 `pnpm check`, Flutter 패키지는 각각
`flutter analyze`를 실행한다. CI도 같은 단위 테스트 진입점을 사용한다.

## 기본 스위트 기준

- `Describe`: 검증할 함수, 클래스 또는 정책을 적는다.
- `Context`: 입력 조건이나 상태를 한 가지로 제한한다.
- `It`: 관찰 가능한 결과 한 가지를 서술한다.
- 현재 시각은 고정하고, 네트워크·데이터베이스·기기 I/O는 사용하지 않는다.
- 테스트 더블은 핵심 분기를 격리하는 데 필요한 최소 범위에서만 사용한다.
- 화면 렌더링, 라우터 이동, 타이머 애니메이션, 전체 API 수명주기, 고정 시나리오 픽스처
  정합성은 기본 단위 스위트에 넣지 않는다.

여러 화면이나 실제 어댑터를 함께 검증해야 하는 시나리오는 단위 테스트와 분리된
별도 통합/E2E 스위트가 생길 때만 추가한다.
